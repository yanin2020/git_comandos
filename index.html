Comandos de GIT más comunes:

    • pwd:" /c/Users/Laptop Win7/desktop/proyecto" Muestra la carpeta donde estamos ubicados
      ls : Para saber el contenido de un folder o carpeta
      cd : para cambiar de ruta, osea me puedo meter en escritorio o desktop y luego en proyecto con ese comando.
      git init :Para inicializar por primera vez un repositorio
      git add nombre del archivo :Para agregar el archivo al staging Area, se puede hacer git add espacio y un punto y agrega todo sin necesidad de escribirlos archivos uno por uno.
      git commit -m "nombre del commit :Sirve para hacerle un snapshot o foto a el estado actual del archivo.
      git log :sirve para ver el hash o cédula de cada commit, hora, día, nombre del desarrollador etc.
      git status :Sirve para ver el estado actual del repositorio.
      git diff nombre_archivo:Sirve para ver los cambios que ha experimentado el archivo, lo que quita lo pone en color rojo y lo que agrega en color verde.
      git restore:Sirve para regresar en el estado del tiempo anterior del repositorio, es como una pequeña máquina del tiempo.
      .git ignore :Sirve para esas carpetas que no estemos utilizando y queremos ignorar en el proyecto.
      clear : Limpia la pantalla.
      git commit :sin el nombre del archivo abre el VIM o shell de linux y para poder editar , tenemos que teclear la letra i, luego escribir el nombre del commit y para que lo acepte le damos escape , luego pegado escribimos :wq pero eso hace más trabajoso el proceso de crear commits, por eso es mejor escribirlo así: git commit -m"mi primer commit"
      git branch: Sirve para ver en que rama estamos del repositorio.
      git branch nombre:archivo :Crea un clon de la rama principal. ejemplo git branch login
      git checkout: Sirve para cambiar de nombre o posición de la rama que queremos utilizar. ejemplo quiero pasar a la rama master y estoy en login, entonces escribimos git checkout master.
